# nRF24L01+PA+LNA

> The 2.4 GHz radio that powers every Bluetooth, BLE, WiFi, and
> drone attack in PhantomRF.

The nRF24L01+ is a single-chip 2.4 GHz transceiver made by Nordic
Semiconductor. The "PA+LNA" variants add an external power
amplifier (PA) on the TX chain and a low-noise amplifier (LNA) on
the RX chain, raising the effective output power to +20 dBm (100
mW) and the RX sensitivity to -105 dBm.

The chip is older — it shipped in 2008 — but it's the cheapest
2.4 GHz radio you can buy and it has a documented
"constant-carrier" mode that doesn't exist on WiFi or BLE chips.
That mode is the foundation of every 2.4 GHz attack in PhantomRF.
See [nrf24-const-carrier.md](../algorithms/nrf24-const-carrier.md)
for the technical deep dive.

## Module variants

| Variant | Output power | RX sensitivity | Range (LOS) | Notes |
|---|---|---|---|---|
| nRF24L01+ (bare) | 0 dBm | -94 dBm | 10–30 m | No antenna connector; PCB trace antenna. |
| nRF24L01+PA+LNA | +20 dBm | -105 dBm | 100–1000 m | External antenna via U.FL or SMA. |
| nRF24L01+PA+LNA-SMA | +20 dBm | -105 dBm | 100–1000 m | Variant with screw-on SMA antenna. |
| nRF24L01+ (clone) | 0 dBm | -94 dBm | 5–10 m | Common on AliExpress; the PA chip is often a relabelled non-PA part. |

PhantomRF **requires** the PA+LNA variant. The bare part's 0 dBm
output cannot saturate a WiFi receiver from a useful distance.

> :warning: **Counterfeit detection.** A genuine PA+LNA module has:
> - A shield can over the RF section
> - An external antenna connector (U.FL or SMA)
> - A 2-amplifier visible inside the shield
> - A heat-sink tab on the PA chip
>
> A counterfeit is just a bare nRF24L01+ in a fancy case. Buy from
> a reputable seller with a 30-day return policy.

## Pinout

```
  ┌────────────────────────┐
  │   nRF24L01+PA+LNA      │
  │                        │
  │  ┌──[shield]──┐        │
  │  │ nRF24L01+  │        │
  │  │   +PA chip │        │
  │  │   +LNA     │        │
  │  └──[shield]──┘        │
  │                        │
  │  ANT  U.FL/SMA         │
  │   │                    │
  │  ╔══╝                    │
  │  ║  [antenna]            │
  │  ╚══╗                    │
  │                        │
  └────────────────────────┘
   Pin layout (2×4 header, looking at the bottom):

   GND  VCC  CE  CSN
   GND  VCC  SCK MISO  MOSI  IRQ
   (note: many breakouts use 2×4 with 8 pins total; the
    second row may be labelled differently — check your board)
```

Standard pinout (looking at the bottom of the module, antenna
facing away):

| Pin | Function | Direction | Notes |
|---|---|---|---|
| GND  | Ground | — | Common ground |
| VCC  | Power | Input | 3.3 V only; absolute max 3.6 V |
| CE   | Chip Enable | Input | HIGH to enter TX/RX; LOW to standby |
| CSN  | Chip Select (active low) | Input | Pull HIGH to deselect the SPI bus |
| SCK  | SPI clock | Input | Up to 10 MHz |
| MISO | SPI MISO | Output | |
| MOSI | SPI MOSI | Input | |
| IRQ  | Interrupt (active low) | Output | Optional; tied to status flags |

## Power requirements

| State | Current | Notes |
|---|---|---|
| Power-down | < 1 µA | CE held LOW for > 10 ms |
| Standby-I | 26 µA | |
| Standby-II | 320 µA | |
| RX (2 Mbps) | 13.5 mA | |
| TX (0 dBm) | 11.3 mA | Bare nRF24, no PA |
| TX (+20 dBm, PA+LNA) | 115 mA peak | The PA-LNA adds ≈ 100 mA |

The PA chip on the breakout module is powered by a separate
on-board switching regulator, which is why the breakout has the
"VIN" pin on some variants — that pin feeds the regulator and
can take 5–7 V. The "VCC" pin is the regulated 3.3 V output and
is what you should connect to the ESP32's 3V3 rail.

> :warning: **Decoupling matters.** The PA-LNA's switching supply
> injects significant noise on the 3V3 rail. Place a **100 nF
> ceramic** in parallel with a **10 µF electrolytic** across the
> module's VCC and GND, as physically close to the module as
> possible. Without this, you'll see SPI errors and
> intermittent "radio not responding" messages.

## Antennas

The antenna is the single biggest factor in range. A bad antenna
can turn a +20 dBm radio into a 0 dBm radio.

| Antenna | Gain | Notes |
|---|---|---|
| PCB trace (on bare nRF24) | 0 dBi | Bad. Don't use. |
| Rubber duck (bundled) | +2 dBi | OK for indoor testing. |
| Whip (1/4 wave monopole) | +2 to +3 dBi | OK for indoor. |
| 1/2 wave dipole | +3 dBi | Better, but needs a ground plane. |
| Yagi (log-periodic) | +8 to +12 dBi | Directional. Long range. |
| Patch (panel) | +6 to +10 dBi | Semi-directional. |
| Helical | +5 to +8 dBi | Circular polarisation. Good for satellites. |

The PA-LNA module ships with a U.FL connector. The bundled
"rubber duck" antenna is a 1/4-wave monopole with a U.FL plug. It
performs fine for indoor testing at 5–10 m. For outdoor use, get a
proper 1/2-wave dipole or a Yagi.

> :warning: **Don't run a PA-LNA without an antenna.** The PA's
> VSWR will rise, the reflected power will heat the chip, and
> you'll burn it out within seconds. The transmit path is also
> mismatched without a 50 Ω load, which can overdrive the LNA on
> RX.

## Range

The nRF24L01+PA+LNA is spec'd to transmit at +20 dBm and receive
at -105 dBm. The link budget is therefore +125 dB. At 2.4 GHz,
free-space path loss is:

```
FSPL(dB) = 20 * log10(d_m) + 20 * log10(f_MHz) - 27.55
```

For 2.4 GHz, FSPL ≈ 100 dB at 100 m. With a +20 dBm PA and a
-105 dBm receiver, the link should just barely close at 100 m
in clear air. In practice, you'll see reliable 30–50 m indoors
(through a wall or two) and 100–300 m outdoors with line of
sight.

For PhantomRF's attacks, the relevant question is **how close
does the target need to be?**:

- **WiFi deauth:** the deauth frame is sent at +20 dBm, so any
  WiFi client within the deauth frame's range (typically 10–30 m
  indoors, 50–100 m outdoors) will receive it. The deauth
  doesn't need to be received by the AP — the client processes
  it independently.
- **BLE spam:** the BLE advertisement is sent at +20 dBm. iOS
  and Android will pop the notification on any device within
  ≈ 30 m.
- **Constant-carrier jam:** the carrier saturates receivers on
  (or near) the channel. Range is essentially the same as the
  nRF24's TX range — 30 m indoors.

## Common failure modes

| Symptom | Likely cause | Fix |
|---|---|---|
| "nRF24 not responding" at boot | Wrong wiring, bad solder joint | Re-check the SPI connections |
| Intermittent "TX failed" | Power supply sags during TX | Add decoupling caps; beef up the 3V3 LDO |
| Short range, no PA heating | Counterfeit module | Buy from a different seller |
| Range drops over time | Antenna connector loose | Re-seat the U.FL connector (it clicks) |
| All-zero RSSI on every channel | CE never goes HIGH | Check the CE pin and its trace |
| SPI errors during heavy attacks | Bus contention | Add a series 10 Ω resistor on each MISO line |
