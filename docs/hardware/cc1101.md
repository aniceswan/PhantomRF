# CC1101

> The sub-GHz radio for 433 / 868 / 915 MHz attacks, recording, and
> replay.

The CC1101 is a sub-1 GHz transceiver made by Texas Instruments.
It supports a wide range of modulation formats (OOK, ASK, GFSK,
4-FSK, MSK) and a frequency range of 300–348 MHz, 387–464 MHz,
and 779–928 MHz. The "green PCB" modules sold everywhere are
based on TI's reference design and come in three frequency
variants (433 MHz, 868 MHz, 915 MHz).

The CC1101's "async-serial" mode is the foundation of every
sub-GHz attack in PhantomRF. It bypasses the chip's packet
handler so the raw demodulated bitstream comes out of GDO0
(RX) or the chip transmits whatever you bit-bang on GDO0 (TX).
See [cc1101-async-serial.md](../algorithms/cc1101-async-serial.md)
for the deep dive.

## Module variants

PhantomRF's CC1101 driver supports any module that uses the TI
reference design. The most common breakouts are:

| Module | Frequency | Antenna | Notes |
|---|---|---|---|
| Green PCB (433 MHz) | 433.05–434.79 MHz | Spring | Most common; ideal for European keyfobs |
| Green PCB (868 MHz) | 868–870 MHz | Spring | For European ISM devices |
| Green PCB (915 MHz) | 902–928 MHz | Spring | For US/AUS ISM devices |
| Red PCB (315 MHz) | 315 MHz | Spring | Older US keyfobs |
| TI CC1101DK | 315/433/868/915 | SMA | Official dev kit; expensive |

> :warning: **The frequency is fixed at order time** on most
> modules. The 433 MHz version is the cheapest and most widely
> available. If you live in an ITU Region 1 country (Europe,
> most of Asia), 433 MHz is the right choice. If you're in an
> ITU Region 2 country (Americas), 915 MHz is more useful for
> LoRa and industrial devices, but 433 MHz still works for
> older keyfobs.

## Pinout

Standard 2×4 header (looking at the bottom of the green PCB):

```
   ┌──────────────────────────────┐
   │        CC1101                │
   │                              │
   │   ANT (spring antenna)  ┌──┐ │
   │                         │  │ │
   │   ┌──────────────────┐  │  │ │
   │   │  TI CC1101 die   │  └──┘ │
   │   │                  │       │
   │   └──────────────────┘       │
   │                              │
   │   GND  VCC  CSN  SCK         │
   │   MISO MOSI GDO0 GDO2        │
   │                              │
   └──────────────────────────────┘
```

| Pin | Function | Direction | Notes |
|---|---|---|---|
| GND  | Ground | — | Common ground |
| VCC  | Power | Input | 3.3 V only; absolute max 3.9 V |
| CSN  | Chip Select (active low) | Input | Pull HIGH to deselect SPI |
| SCK  | SPI clock | Input | Up to 9 MHz (CC1101 limit) |
| MISO | SPI MISO | Output | |
| MOSI | SPI MOSI | Input | |
| GDO0 | General Digital Output 0 | Configurable | Used as the async-serial data line |
| GDO2 | General Digital Output 2 | Configurable | Used as the FIFO threshold indicator |

GDO0 is the workhorse — it's the data line in async-serial mode.
GDO2 is optional; it's set up as a FIFO threshold in M1.

## Frequency bands

The CC1101 has three main frequency ranges, with **two gaps**:

| Band | Start | Stop | Notes |
|---|---|---|---|
| Low band | 300 MHz | 348 MHz | European 315 MHz keyfobs, some industrial |
| Mid band 1 | 387 MHz | 464 MHz | 433 MHz ISM, 450 MHz industrial |
| Mid band 2 | 779 MHz | 899.99 MHz | 868 MHz ISM, 900 MHz LoRa |
| High band | 900 MHz | 928 MHz | US 915 MHz ISM, LoRaWAN |

> :warning: **There is no usable range from 348–387 MHz or
> 464–779 MHz.** The PA simply cannot hit those frequencies
> efficiently. This is a TI design constraint, not a firmware
> bug.

The frequency is set by the 24-bit FREQ register:

```
FREQ = (freq_Hz * 65536) / 26_000_000
```

For 433.92 MHz:

```
FREQ = (433_920_000 * 65536) / 26_000_000
     = 1_093_856
     = 0x10B0E0
```

So `FREQ2 = 0x10`, `FREQ1 = 0xB0`, `FREQ0 = 0xE0`. The firmware
has a helper for this in
[channel-math.md](../algorithms/channel-math.md).

## Registers worth knowing

The CC1101 has about 50 configuration registers. Most of them
are set by the library; here are the ones you might want to tweak
by hand.

| Register | Address | Purpose | Default |
|---|---|---|---|
| IOCFG0 (GDO0) | 0x02 | Configure GDO0 function | 0x06 (sync word) |
| PKTCTRL0 | 0x08 | Packet format control | 0x05 (CRC + variable length) |
| FREQ2/1/0 | 0x0D/0x0E/0x0F | Frequency | varies |
| MDMCFG2 | 0x12 | Modem config (modulation, sync) | 0x02 (2-FSK) |
| DEVIATN | 0x15 | Frequency deviation | 0x15 |
| FREND0 | 0x22 | Front-end TX config | 0x10 |
| PA_TABLE0 | 0x3E | PA power table (one byte per band) | varies |

The `SmartRC-CC1101-Driver-Lib` library handles register writes
through higher-level functions like `setFrequency()`, `setModulation()`,
and `setOutputPower()`. Direct register access is reserved for
the async-serial mode in the M0 firmware.

For async-serial mode specifically:

```cpp
// 1. Switch to async-serial mode
radio.setCCMode(0);          // 0 = async-serial, 1 = normal packet
radio.setPktFormat(3);       // 3 = asynchronous serial mode
radio.setLengthConfig(0);    // ignore length byte
```

This disables the chip's packet handler and routes the demodulated
data to GDO0.

## Power requirements

| State | Current | Notes |
|---|---|---|
| Power-down (SPWD) | < 1 µA | Sleep mode |
| Idle | 1.7 mA | Ready to RX or TX |
| RX (433 MHz) | 15 mA | |
| RX (868/915 MHz) | 16 mA | |
| TX (+12 dBm) | 30 mA | Maximum PA setting |
| TX (0 dBm) | 15 mA | Minimum PA setting |

The CC1101 is much more power-efficient than the nRF24L01+PA+LNA
because the PA is internal and the output power is lower
(+12 dBm max vs. +20 dBm). The trade-off is range: at +12 dBm,
a typical link budget is 110 dB, giving 30–100 m outdoors.

For battery-powered builds, the CC1101 is the friendlier of the
two radios.

## Antennas

The bundled spring antenna is a 1/4-wave monopole. It's fine for
indoor testing at 5–15 m, but for serious work, get a proper
1/2-wave dipole or a 5/8-wave whip.

| Antenna | Gain | Notes |
|---|---|---|
| Spring (bundled) | +2 dBi | Indoor, short range |
| Whip (1/4 wave) | +2 to +3 dBi | Indoor, better ground plane needed |
| 1/2 wave dipole | +3 dBi | Outdoor line of sight |
| 5/8 wave whip | +5 dBi | Outdoor, longer range |
| Yagi | +8 to +12 dBi | Outdoor, directional |

> :warning: **Antenna length matters more at sub-GHz.** A 433 MHz
> 1/4 wave is 17 cm; a 915 MHz 1/4 wave is 8 cm. The bundled
> spring is sized for the module's frequency variant — if you
> bought a 433 MHz module but the spring is for 915 MHz, the
> SWR will be high and range will suffer.

## Common failure modes

| Symptom | Likely cause | Fix |
|---|---|---|
| "CC1101 not found" at boot | Wrong wiring, bad SPI mode | Check SCK/MISO/MOSI/CSN; ensure CSN is HIGH at boot |
| Reads back 0x00 or 0xFF for every register | SPI not initialised, or MISO floating | Check the MISO pull-up; check SPI clock polarity |
| Records noise, no data | Wrong frequency or modulation | Verify the target device's frequency with an RTL-SDR |
| Transmit power very low | PA table not configured for the band | Re-calibrate with `calibrate()` after every frequency change |
| Range much shorter than expected | Antenna mismatch or wrong band | Check the spring antenna length matches the module's frequency |
